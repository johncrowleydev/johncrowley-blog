One of the most interesting shifts in software work is the ability to orchestrate real development from a phone.

The key is not trying to do everything in the phone browser. The key is using the phone as a control surface while remote tools do the heavier work.

## What matters most

1. Fast iteration loops
2. Clear task delegation
3. Small, reliable deployment targets
4. Good defaults for publishing and rollback

For a personal site, a static build on GitHub Pages is a great fit because it is simple, cheap, and robust.
